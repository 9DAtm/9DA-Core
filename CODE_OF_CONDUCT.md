# Code of Conduct

## Our Commitment

The 9DA project is committed to providing a welcoming, professional, and inclusive environment for all contributors, regardless of background or identity.

This Code of Conduct applies to all project spaces, including GitHub repositories, discussions, issues, and any communication related to the project.

---

## Expected Behavior

### Professional Standards

- Use clear, respectful communication
- Focus on structural and technical merit
- Provide constructive feedback
- Respect different viewpoints and experiences
- Accept constructive criticism gracefully
- Prioritize ecosystem integrity over personal preferences

### Governance Alignment

- Follow the ADR process for structural proposals
- Respect architectural invariants
- Uphold ethical constraints
- Maintain audit trail integrity
- Report violations appropriately

---

## Unacceptable Behavior

The following behaviors are not acceptable:

- Harassment, intimidation, or discrimination of any kind
- Personal attacks or ad hominem arguments
- Publishing others' private information without permission
- Trolling, insulting/derogatory comments, or personal attacks
- Sustained disruption of discussions
- Advocating for or encouraging any of the above behaviors

### Governance-Specific Violations

- Circumventing architectural constraints
- Knowingly introducing ethical violations
- Forging compliance claims or certifications
- Retaliating against violation reporters
- Pressuring maintainers to compromise integrity

---

## Scope

This Code of Conduct applies:

- Within project repositories
- In GitHub Discussions and Issues
- In pull request reviews and comments
- When representing the project publicly
- In any space where project business is conducted

---

## Enforcement

### Reporting

Violations can be reported:

**Public violations:**
- GitHub Issues with `code-of-conduct` label
- Public discussion threads

**Sensitive violations:**
- Email: zdenkacucin@gmail.com
- Direct message to @9DAtm
- Anonymous via designated third party (TBD)

### Investigation

All reports will be:
- Reviewed promptly
- Investigated thoroughly
- Handled confidentially where appropriate
- Documented in audit trail

### Response Actions

Depending on severity:

**Minor violations:**
- Private warning
- Request for public apology
- Temporary suspension from discussions

**Serious violations:**
- Temporary ban from repository access
- Public warning
- Revocation of merge rights

**Severe violations:**
- Permanent ban from ecosystem
- Public notice
- Revocation of all certifications
- Referral to appropriate authorities if necessary

---

## Appeals

Appeals may be submitted:

- Within 14 days of enforcement action
- Via email to zdenkacucin@gmail.com
- With new evidence or clarification
- One appeal permitted per violation

Appeals will be reviewed by:
- Different reviewer than initial enforcer
- Final decision within 14 days
- Decision is final
